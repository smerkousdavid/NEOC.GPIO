Detailed Description

Handler for the builtin gyro.

This class will handle the builtin gyro and has a builtin calibrator. The calibrator is extremely simple and not precise, but it does its job. If anyone would like to help me write a better one, that would be awesome.


Here is the example usage of the class:

int main() {
Gyro::init(true); //Throw errors if any
Gyro::setPoll(30); //Set the update rate to 30 milliseconds
Gyro::calibrate(100, 30); //3 seconds calibration (100 samples at 30 millis delays)
while(1) {
int x, y, z; //Hold the gyro vals
Gyro::read(x, y, z); //Update the values
printf("X: %d, Y: %d, Z: %d\n", x, y, z); //Print the updates
usleep(1000 * 30); //30 millis delay
}
return 0; //Auto free on exit
}

inlinestatic

Setting the poll rate of the Gyro.

Usually you will probably run your code within some type of loop that runs on forever. Well obviously that you are probably using the gyro you are using a loop. Check the delay on that loop and set it to the rate below! If you make the pull too fast the gyro will reset its values, if you pull too slow then the values you pull might be the same when you pull again. Try making the value slightly higher than what you pull, it will save you some processing.

Example Usage:

Gyro::setPoll(30); //Set poll rate for 30 milliseconds
Returns
A boolean if the operation succeded or not
Parameters
millisThe milliseconds until every read in your loop
throwsOptional value to throw if there is an error (default: true)
Note
To get the best results set the millis to a number above 50 millis since
The number resets to 0 every new data sample. So small numbers means fast update but small values

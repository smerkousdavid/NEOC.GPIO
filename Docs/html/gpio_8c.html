Detailed Description

GPIO controls and pinouts, extremely fast and is used for fake pwm.

Author
David Smerkous
Date
11/28/2016 This is the source of the gpio controllers for both the inner and outer banks of the Udoo Neo. Just to be safe make sure the m4 core is disabled.
See also
neo_disable_m4()
Note
Please make sure the m4 core is disabled so that you don't ruin the board
If you are advanced and still want to use m4, make sure you don't set the same pins to output

int neo_gpio_attach_interrupt ( int  pin,
const char *  mode,
interruptfunc  intfunc

Attaches an interrupt to a pin.

This will start a new thread that listens for a pin change state based on what you want A state of either "rising" to only detect when the pin changes from 0 to 1. "falling" for the function to only call from 1 to 0 and "both" that happens if it changes either way.

Returns
NEO_OK/NEO_INTERRUPT_ERROR/NEO_PIN_ERROR/NEO_UNUSABLE_ERRROR if the interrupt failed
Parameters
pinThe pin to attach the interrupt to
modeThe mode to put the pin in available ("both", "rising", "falling")
intfuncThe function pointer to the interrupt return

§ neo_gpio_digital_read()

int neo_gpio_digital_read ( int  pin)

Reads either LOW or HIGH from gpio pin.

This will read the boards current state. If the number is a negative number then the result has failed. Either than that the result will either return a LOW or HIGH value. Same as arduino.

Returns
NEO_OK or NEO_READ_ERROR/NEO_PIN_ERROR/NEO_UNUSABLE_ERROR if some GPIO read failed
Note
You must call neo_gpio_pin_mode(<pin>, INPUT); before reading
Warning
Do not use 5v with these boards! It will ruin the board

int  direction

Writes either LOW or HIGH to gpio pin.

This will set the board pin to either a low or high state. Low will be 0 volts and high will be roughly 3.3v. Since this board works on 3.3v don't use 5v as an input!

Returns
NEO_OK or NEO_DIR_ERROR/NEO_PIN_ERROR/NEO_UNUSABLE_ERROR if some GPIO write failed
Note
You must call neo_gpio_pin_mode(<pin>, OUTPUT); before writing
Warning
Do not use 5v with these boards! It will ruin the board

§ neo_gpio_free()

int neo_gpio_free ( )

Releases the gpio pins from program.

This is the exact opposite of neo_gpio_init where you deinitialize the pins Freeing up the pins for another process to use

Returns
NEO_OK or NEO_UNUSABLE_ERROR if a pin failed to release (most likely will happen)

§ neo_gpio_init()

int neo_gpio_init ( )

Initializes the GPIO controller.

Starts the gpio pins all having the direction of in and are tested against each other to see if the pin is available. For a fast usable gpio list. This can be called multiple times safely.

Returns
NEO_OK or NEO_EXPORT_ERROR/NEO_UNUSABLE_ERRROR if some GPIO weren't initialized
Note
Don't use the same GPIO pin as PWM although they're mapped differently don't do it
NEO_UNUSABLE_ERROR might return if it's already in use or PWM is mapped to it

§ neo_gpio_pin_mode()

int neo_gpio_pin_mode ( int  pin,
int  direction 
)

Sets the direction of the pin.

Sets the direction of the pin to either two states INPUT OR OUTPUT if it's not either one Of those a NEO_DIR_ERROR will be returned. If teh pin is not a valid pin then it can return either a NEO_PIN_ERROR (Out of range) or a NEO_UNUSABLE_ERROR (Pin not valid or is in use by another component)

Returns
NEO_OK/NEO_DIR_ERROR/NEO_PIN_ERROR/NEO_UNUSABLE_ERRROR if some GPIO weren't initialized
Note
Don't set to OUTPUT unless you know for sure the m4 (arduino) core isn't using the same pin for OUTPUT
NEO_UNUSABLE_ERROR might return if the pin is unusable

Detailed Description

The gpio class that handles all General Pin Input and Output.

This class is a helper to use the General Pin Input and Output. Just remember it's faster to use the C methods statically rather than the objects and new instances.


Here is the example usage of the class:

int main() {
Gpio led(13);
led.setOut();
led.on();
return 0;
}


Note
The m4 core DOESN'T have to be disabled, you just have to be very careful about what pins you use

Gpio pin object deconstructor and pin release.

This will attempt to release the objects, if specified in the constructor that when all objects are out of scope and destroyed to release all of the pins from the program

Note
Do not try to mix Object oriented and static usage of any Neo Object it screws it up
When attempting to use both static Gpio::...() and obj....() make sure you use Gpio obj(<pin>, false) to not release on 0 objects
When using static :: and object . at the same time, you use false on release on the object constructor and must call Gpio::free() at the end of the program

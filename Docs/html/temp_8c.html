Detailed Description

Control the temperature brick I2C plug in sensor.

Author
David Smerkous
Date
11/28/2016 This is the source file to control the I2C plug in temperature not barometric brick sensor using the system default I2C temp driver via sysfs. This is all used on I2C - 2. If you want to use it as regular PWM, look at the device tree editor or if you want to use it as plain I2C look at the (soon to be, if it's up, tell me to remove this message) I2C functions.

§ neo_temp_init()

int neo_temp_init ( )

Temperature brick initializer.

This will reload and load the I2C driver for the temperature brick. Then attach the program to that driver and interface. Make sure this program is ran as ROOT and not just sudo

Returns
NEO_OK or NEO_UNUSABLE_ERROR if temp brick is not found
Note
NEO_UNUSABLE_ERROR means that there isn't a temp module/brick detected
Warning
This would be way better if ran as ROOT rather than SUDO like so sudo su -c './a.out'

int neo_temp_read ( )

Temperature reading.

The main read method which will return the current temperature reading in milli celcius

Returns
A milli celcius value or a NEO_UNUSABLE_ERROR if temp brick is not found
Note
To get celcius just divide by one thousand (make sure you make the int a float)

Detailed Description

Most commonly thrown error when the specified pin is unusable.

This one can be very problematic because it could be sometimes hard to find what the issue is. Most of the time it's because of these issues

  • PWM pin wasn't properly pinmuxed using device tree editor
  • The PWM pin isn't the same mapping as it is in the device tree editor trust me it is usually not
  • GPIO is already using the same bank pin where PWM is trying to use
  • Same as above but vise versa with GPIO and PWM
  • You don't have suffice permissions, try running as root
  • The OS just plain won't let you allocate that pin, probably meaning you havent disabled the m4 core yet


As soon as you have tried fixing all of those issues or at least checking for them. Try emailing me at smerk.nosp@m.ousd.nosp@m.avid@.nosp@m.gmai.nosp@m.l.com to see if I can try to help you out. I understand this error can be extrememly annoying.

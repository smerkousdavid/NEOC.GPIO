Detailed Description

Thrown when for some dumb reason the program failed to write or read from a pin.

If you haven't recieved a UnusableError and still got a ReadWriteError. It usually means if you at the EXACT same time try to write to a pin, then flush (in the backend) flush the new settings. It will throw this error, either than that you should rarely ever see this.
